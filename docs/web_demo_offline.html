Modbus Explorer

Read raw register values directly from any Modbus slave device connected to the RS485 bus.

Write Operations
Caution: Writing Modbus registers directly can affect device operation. Ensure you are writing to the correct register and slave device before proceeding.

Write Single Register (FC 06)

Write a single 16-bit value to a holding register on the specified slave device.

Write Multiple Registers (FC 16)

Write multiple consecutive 16-bit registers. Enter values separated by commas.

Modbus Bus Scan

Scan the RS485 bus to discover active slave devices. Scans slave IDs 1 through 247.

Trigger OTA Update

Enter the URL to a binary firmware file (.bin) hosted on GitHub Releases or an accessible HTTPS server. The device will download, verify, and apply the update automatically.

Important: The device will reboot after a successful update. Ensure the firmware file is valid and compatible with this hardware revision. MQTT will be disconnected during the update to free heap memory.

Release Notes — v1.3.8

Memory Optimizations (v1.3.8)

  • Max sensors reduced from 15 to 10 (saves ~9KB heap)
  • Task stack sizes optimized across modbus, telemetry, modem tasks
  • Static buffer sizes reduced for MQTT URIs and topics
  • Eliminated heap fragmentation in batch telemetry assembly

Reliability Fixes (v1.3.8)

  • SD card I/O retry logic with 3 attempts + RAM buffer fallback
  • Telemetry timeout no longer triggers restart when SD card is caching
  • SPI bus properly freed on SD card mount failure
  • Fixed wrong timestamps on SD card cached messages
